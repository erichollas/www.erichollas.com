My B.S. degree was in Mathematics and a minor in Physics from The University of Texas at San Antonio. I also took a few graduate classes and did some work towards a thesis in the Mathematics department and Physics department at Texas Tech University.

To put it simply, Calculus III inspired most of my time at these schools. My thesis, by the direction of my professor's research area, was going to be on the properties and use of isometries between hyperbolic surfaces. The purpose of this research was to re-parametrize complicated surfaces that are not integrable to be integrable and maintain their geometric properties, particularly surfaces arising in partial differential equations. This mathematics research has implications in Relativistic, Lagrangian Dynamics, and the Calculus of Variations. Unfortunately I was unable to finish this thesis.

Games in Vulkan

Vulkan Render Engine

The Vulkan Render Engine is a simple rendering engine (well, simple for Vulkan). The overall goal of this project was to make something reusable. One way I saw to accomplishing this was to make an object that acted as an interface between the render engine and whatever program implements it. This object is the GeometryManager class (Not a singleton object). This object is a vector containing the vertices and indices to be loaded into vertex and index buffers and all of the offset locations for each object within the vertex and index buffers. This object will be passed by reference to the render engine object in any implementation, and the manager will be used to create and re-create the vertex buffer, index buffer, the offsets for the dynamic uniform buffer, and the drawing-by-index function in the command buffer object.

The source code for this project can be found on my GitHub.





Multi-threaded Minesweeper

As far as game mechanics, this clone of minesweeper will follow the classical game of minesweeper. This game is an instance of a game that implements the Vulkan Render Engine.

The overall goal of this project was to implement multi-threaded algorithms in a game setting. I was inspired by a textbook called C++ Concurrency in Action: Practical Multithreading, by Anthony Williams to design thread safe functions. This book goes much deeper than the usual issues that arise in multi-threading programming like data races, and it is also mindful of the memory space needed for each of the threads. Thus, this project made for a good exercise in implementing multi-threaded algorithms with regards to thread safety as well as each threads’ memory space.

The source code for this project can be found on my GitHub. Where to find the callable function, two lambdas, and the rest of the multi-threaded code will be listed on the associated readme file for the project.





State Machine 3D-PONG

The game logic in this clone of PONG is the same from the classical game of PONG. This version is presented in 3D and is another game to implement the Vulkan Render Engine. However, the main goal of this project is to implement FSM's that communicate with one another. Unnecessary for pong.... perhaps, but an exercise nonetheless.

The design of the FSMs follow the design described in chapter 2 of Mat Buckland's game AI text book, which can be found here. The character classes inherit from a base class and contain a StateMachine object as a class member. The StateMachine object is a template class that runs the runCycle() method, or update() method, for their respective class object. The StateMachine object also handles the messages sent between characters via the Messanger, a static singleton class. The states are singletons that inherit from a base template class, and the states define the execute methods and onMessage events. In this implementation, the onMessage events are collisions between the paddle and ball characters and the score changes. All of these characters are added into the ChraracterManager object, which acts as the data structure of the characters. The CharacterManager will keep track of the uniform matrices for each character and run runCycle() function each pass through the main game loop.

The source code for this project can be found on my GitHub.
