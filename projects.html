Vulkan Render Engine

Source code found here on GitHub.

The Vulkan Render Engine is a simple rendering engine (well, simple for Vulkan). The overall goal of this project was to make something reusable.

One way I saw to accomplishing this was to make an object that acted as an interface between the render engine and whatever program implements it. This object is the GeometryManager class (Not a singleton object).

This GeometryManager is a vector containing the vertices and indices to be loaded into vertex and index buffers and all of the offset locations for each object within the vertex and index buffers. This object will be passed by reference to the render engine object in any implementation, and the manager will be used to create and re-create the vertex buffer, index buffer, the offsets for the dynamic uniform buffer, and the drawing-by-index function in the command buffer object.





Multi-threaded Minesweeper

    Source code found here on GitHub.

    Done using C++ and the previously mentioned Vulkan Render Engine.

    Clone of the classic minesweeper.

    Multi-threaded algorithms are used to populate the board, calculate the number of adjacent mines, and reveal multiple tiles when clicked.

    Check out the readme on my GitHub for the locations of where to find the code that is multithreaded.

    In terms of of multithreading, not only was I concerned with performance, but I focused on how each thread creates its own memory and how to manage that memory.

    This project to inspired me to study further on multithreading. I enjoy it greatly.
